🎯 ¿QUÉ SE HIZO?

  Tu API-One ahora funciona con "Form 1: Auto-Fallback"
  
  Significa:
  ─ OneCode/VSCode ve UN SOLO modelo: "api-fallback"
  ─ Internamente, tu API intenta Groq → si falla, Gemini
  ─ Todo completamente transparente para el usuario

📊 EJEMPLO DE FLUJO

  OneCode envía:
  ┌────────────────────────────────┐
  │ model: "api-fallback"          │
  │ messages: "Hola, ¿cómo estás?" │
  └────────────────────────────────┘
           ↓
  Tu API internamente:
  ├─ model = 'auto'
  ├─ Intenta GROQ → ✓ Éxito → Retorna
  └─ Si falla → Intenta GEMINI → ✓ Éxito → Retorna
           ↓
  OneCode recibe:
  ┌────────────────────────────────────┐
  │ model: "api-fallback"              │
  │ response: "Hola, estoy bien!"      │
  │ metadata: {provider: "groq"}       │
  └────────────────────────────────────┘

  ¡Usuario nunca sabe qué pasó internamente!

  P: ¿Cómo sé qué proveedor se usó?
  R: Mira metadata.provider en la respuesta

  P: ¿Puedo cambiar el orden?

  P: ¿Hay límite de requests?
  R: Sí, límites gratuitos de cada proveedor

  P: ¿Es seguro?
  R: Sí, usa autenticación por API key
