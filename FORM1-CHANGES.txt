🎯 COMPORTAMIENTO RESULTANTE
─────────────────────────────────────────────────────────────────────────────

ANTES (Multiple Models):
  OneCode ve 5+ modelos → Usuario elige uno → Fallback depende de su elección

AHORA (Form 1 - Auto-Fallback):
  OneCode ve 1 modelo (api-fallback) → Usuario NO elige → Fallback SIEMPRE activo

FLUJO TRANSPARENTE:
  OneCode: "Usa api-fallback"
  ↓
  Tu API: "Vale, internamente voy a usar 'auto' (fallback)"
  ↓
  Intenta Groq primero (rápido, gratuito)
  ├─ ✓ Éxito? → Retorna respuesta
  └─ ✗ Falla? → Intenta Gemini
     ├─ ✓ Éxito? → Retorna respuesta
     └─ ✗ Falla? → Error 503
  ↓
  OneCode recibe: { model: "api-fallback", response: "..." }
  OneCode muestra la respuesta sin saber qué pasó internamente

🧪 PRUEBAS
─────────────────────────────────────────────────────────────────────────────

Para verificar que funciona:

1. Inicia servidor:
   node src/server.js

2. Verifica modelos disponibles:
   curl -X GET http://localhost:3000/v1/models \
     -H "Authorization: Bearer dev-local-key-1"
     
   Debe retornar SOLO "api-fallback"

3. Prueba chat:
   curl -X POST http://localhost:3000/v1/chat/completions \
     -H "Authorization: Bearer dev-local-key-1" \
     -H "Content-Type: application/json" \
     -d '{
       "model": "api-fallback",
       "messages": [{"role": "user", "content": "Hola"}]
     }'
     
   Debe retornar respuesta con metadata mostrando:
   - "model": "api-fallback"
   - "provider": "groq" (o "gemini" si falló groq)

4. Ejecutar script de prueba (opcional):
   node test-form1.js
